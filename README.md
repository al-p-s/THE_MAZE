# The Maze — Dev Plan

## Стек
- Frontend: React + Vite + Canvas
- Backend: Node.js + Socket.io
- БД: PostgreSQL (Docker) — второй этап
- Деплой: Docker Compose

## Структура проекта
```
the-maze/
├── client/
│   └── src/
│       ├── components/
│       │   ├── MazeCanvas.jsx   # изолированный Canvas-рендер
│       │   ├── GameUI.jsx       # HUD: ОД, хп, дебаффы, инвентарь
│       │   ├── ActionPanel.jsx  # кнопки действий, d-pad

    ├── index.js
    └── game/
        ├── maze.js      # генерация лабиринта, POI, спавн

        └── classes.js   # параметры классов (урон, спецабилки)
```

## Текущий класс для разработки
Юджин «Пинкертон» — базовый, без спецмеханик (временно). Остальные 7 — после полной реализации базовой механики.

## Нюансы реализации (важно)
- Стены клетки игрок НЕ видит при заходе. Стены прощупываются отдельным действием или попыткой пройти.
- visibleCells хранится как объект { "x,y": { top, right, bottom, left } }, где true = стена проверена.
- Выход — дырка во внешней стене крайней клетки (не тип клетки). Хранится как { x, y, direction }.
- Выход открывается только после проверки стены или попытки в неё пройти.
- Внешние стены неуязвимы. Без клада выйти нельзя — если только клад не уничтожен.
- Игроки видят друг друга только если находятся в одной клетке. Рендер масштабируется по числу игроков в клетке.
- Canvas масштабируется под размер экрана и размер лабиринта динамически.

## Этап 1 — Серверная логика

### ✅ Реализовано
- Генерация лабиринта (рекурсивный бэктрекинг, полная связность)
- Размер поля зависит от числа игроков (getMazeSize)
- Расстановка POI: Выход (1, дырка во внешней стене), Арсенал (ceil(n/3)), Госпиталь (ceil(n/3)), Клад (1, закопан)
- Спавн игроков — случайный по периметру, без совпадений
- GameState: players, maze, treasure, exit, currentTurn, status
- Туман войны: revealCell, revealWall, getPlayerView
- Система ходов: очередь, ОД, передача хода
- Дебаффы W / S / P с тиком по ходам
- Перемещение: движение, столкновение со стеной, открытие клетки
- Выход: обнаружение через проверку/попытку пройти, knownExit на игроке
- Проверка стены (actionCheckWall) — в т.ч. определяет выход
- Проверка соседней клетки на опасность (actionCheckCell)
- Атака по прямой: бросок кубика, урон, дебафф, остановка у стены/игрока
- Ближняя атака (Melee)
- Падение клада при любом ранении
- Арсенал: +2 патрона или +2 бомбы (рандомно), исчезает после обыска
- Госпиталь: лечение до макс или аптечка, исчезает после использования
- Клад: выкопать, поднять, закопать, падение при уроне
- Бомба: установка мины (1.5 урона), взрыв внутренней стены
- Аптечка: +1 хп
- Смерть игрока, дисконнект
- Конец игры: последний живой, выход с кладом
- cellmates в getPlayerView — игроки в одной клетке

### ⬜ Осталось
- Механики Пинкертона: радар 3x3, бесшумное столкновение, перебросить кубик
- Уничтожение клада (взрывом бомбы/базуки) + вин кондишн побега
- Труп на карте как объект (лут с трупа)
- Звуковые/текстовые подсказки по радиусу (выстрел, взрыв, столкновение)
- Базука (отложено)

## Этап 2 — Фронтенд

- Отрисовка игрока: круг, хелсбар (полукруги для 0.5 хп), индикатор клада

- Выход: голубая дырка в стене, появляется только после обнаружения

- Экраны: ожидание, игра, конец игры
